# Demo

## Example 结构

- 首页：Hero + Features + Components + Install + Theme
- 组件页：标题 + 描述 + demo section
- 侧边栏：基础组件 / 复杂组件分类

## 首页规则

- 使用草地 / 岛屿氛围背景
- 标题要有游戏感，不要普通 SaaS hero
- 组件一览要用 card 网格，不用朴素 list

## 组件页规则

每个 section 建议包含：

- 标题
- tag
- 一段说明
- 演示区域
- 必要时附示例代码
- 必要时附 API 表

如果是 `BottomSheet`：

- 至少提供一个真实触发按钮，而不是只做静态截图
- 同时展示一个嵌入式预览，方便检查底部锚定、圆角和安全区表现
- 示例内容优先是任务说明、操作确认、选择列表这类移动端场景

## 响应式

- `<768`：单列、缩小标题字号
- `768-899`：内容单列，支持 Drawer
- `>=900`：固定左侧导航 + 主内容区

## Phone Demo

- 允许横向滚动
- 不要强制压缩到变形

## CodeBlock Demo

- 保持深色块体对比
- 代码区域需要可横向滚动
